Help & About

About SlipStream GUI

SlipStream GUI is a user-friendly, cross-platform VPN client that makes it easy to connect to SlipStream VPN. It provides secure, encrypted tunneling for all your internet traffic through an intuitive graphical interface.

Whether you're looking to protect your privacy, access geo-restricted content, or secure your connection on public Wi-Fi, SlipStream GUI makes VPN protection accessible to everyone.

How to Use

Step 1: Configure Settings (Optional)

  • Enter your DNS Resolver (default: 8.8.8.8:53)
  • Enter your SlipStream server domain (default: s.example.com)

Step 2: Start the VPN

  • Click the "Start VPN" button
  • Wait for the status indicators to show "Running"
  • The app will automatically configure system proxy if the toggle is enabled

Step 3: Use Your Apps

  • If system proxy is configured, all apps will automatically use the VPN
  • Otherwise, configure individual apps to use HTTP proxy: 127.0.0.1:8080
  • Check the logs panel to see connection activity

Step 4: Stop When Done

  • Click "Stop VPN" when you want to disconnect
  • System proxy will be automatically unconfigured if it was auto-configured

Features

  • Easy Setup: No technical knowledge required - just click Start
  • System-Wide Protection: Route all your traffic through the VPN
  • Real-Time Logs: Monitor connection activity with detailed logs
  • Verbose Logging: Toggle detailed technical logs for debugging
  • Test Connection: Verify your proxy is working with the test button
  • Cross-Platform: Works on both macOS and Windows

Status Indicators

  • SlipStream Client: Shows if the SlipStream process is running in the background
  • HTTP Proxy: Shows if the HTTP proxy server is active (listening on 127.0.0.1:8080)
  • System Proxy: Shows if system proxy is configured (optional - you can configure apps manually)

Settings Explained

  • Configure System Proxy: Automatically configure your system to use the VPN for all apps
  • DNS Resolver: The DNS server to use for domain name resolution (format: IP:Port)
  • Domain: Your SlipStream server domain address
  • Verbose Logging: Show detailed connection logs (useful for troubleshooting)

Note: Changes to DNS Resolver or Domain require restarting the VPN to take effect.

Technical Details

Architecture Overview

SlipStream GUI is built with Electron and uses a multi-layer proxy architecture:

Application Layer (Browser/Apps)
    ‚Üì HTTP/HTTPS Requests
HTTP Proxy Server (Node.js - Port 8080)
    ‚Üì SOCKS5 Protocol
SOCKS5 Client (Port 5201)
    ‚Üì Encrypted Tunnel
SlipStream VPN Server

How It Works

1. SlipStream Client Process:

  • Runs the native binaries/slipstream-client-mac-arm64 / binaries/slipstream-client-mac-intel, binaries/slipstream-client-win.exe, or binaries/slipstream-client-linux binary
  • Establishes SOCKS5 proxy on 127.0.0.1:5201
  • Handles encryption and connection to SlipStream servers

2. HTTP Proxy Server:

  • Built with Node.js http module
  • Listens on 127.0.0.1:8080
  • Handles HTTP requests via socks-proxy-agent
  • Handles HTTPS via CONNECT method with socks library
  • Uses http-proxy library for enhanced CONNECT handling

3. System Proxy Configuration:

  • macOS: Uses networksetup -setwebproxy and networksetup -setsecurewebproxy
  • Windows: Uses netsh winhttp set proxy
  • Automatically detects active network interfaces on macOS

Code Structure

  • main.js: Electron main process - handles window creation, IPC, proxy server, and system integration
  • index.html: Renderer process - UI, status display, logs, and user interactions
  • check-system-proxy.js: Helper module to check current system proxy status
  • package.json: Dependencies: electron, socks, socks-proxy-agent, http-proxy

Key Technologies

  • Electron: Cross-platform desktop app framework
  • Node.js HTTP Module: HTTP proxy server implementation
  • SOCKS5: Protocol for routing traffic through SlipStream client
  • IPC (Inter-Process Communication): Communication between main and renderer processes
  • Child Process: Spawning and managing SlipStream client binary

Request Flow

1. Browser sends HTTP request to 127.0.0.1:8080
2. HTTP Proxy receives request
3. For HTTP: Uses SocksProxyAgent to forward via SOCKS5
4. For HTTPS: 
   - Receives CONNECT request
   - Creates SOCKS5 tunnel via SocksClient.createConnection
   - Sends "200 Connection established" response
   - Pipes raw socket data bidirectionally
5. SOCKS5 client forwards to SlipStream server
6. Response flows back through the same path

Configuration Files

  • settings.json: Stores user preferences (resolver, domain, verbose logging)
  • Created automatically in app directory
  • Persists across app restarts

Sharing PC Internet via Mobile (Same Network)

If your PC and mobile device are on the same Wi-Fi network, you can configure your mobile device to use your PC's internet connection (including the VPN) through the proxy.

Prerequisites:

  • PC and mobile device must be connected to the same Wi-Fi network
  • SlipStream GUI must be running with VPN started
  • Find your PC's local IP address (see instructions below)

Finding Your PC's IP Address:

  • macOS/Linux: Open Terminal and run ifconfig | grep "inet " | grep -v 127.0.0.1 or ip addr show
  • Windows: Open Command Prompt and run ipconfig, look for "IPv4 Address" under your active network adapter
  • It will typically look like 192.168.1.XXX or 10.0.0.XXX

üì± iOS Configuration

  1. On your iPhone/iPad, go to Settings ‚Üí Wi-Fi
  2. Tap the (i) icon next to your connected Wi-Fi network
  3. Scroll down to "HTTP Proxy" section
  4. Select "Manual"
  5. Enter your PC's IP address in "Server" field (e.g., 192.168.1.100)
  6. Enter "8080" in the "Port" field
  7. Leave "Authentication" off
  8. Tap "Save" in the top right

Note: Your iOS device will now route all internet traffic through your PC's VPN connection. To disable, go back to Wi-Fi settings and set HTTP Proxy to "Off".

ü§ñ Android Configuration

  1. On your Android device, go to Settings ‚Üí Wi-Fi
  2. Long-press on your connected Wi-Fi network
  3. Select "Modify network" or "Network details"
  4. Tap "Advanced options" or expand the advanced settings
  5. Under "Proxy", select "Manual"
  6. Enter your PC's IP address in "Proxy hostname" (e.g., 192.168.1.100)
  7. Enter "8080" in "Proxy port"
  8. Leave "Bypass proxy for" empty (or add local addresses like localhost,127.0.0.1)
  9. Tap "Save"

Note: Some Android versions may have slightly different menu paths. If you can't find these options, try: Settings ‚Üí Network & Internet ‚Üí Wi-Fi ‚Üí (tap network) ‚Üí Advanced ‚Üí Proxy.

To disable: Go back to Wi-Fi settings, modify the network, and set Proxy back to "None".

‚ö†Ô∏è Important:

  • Make sure your PC's firewall allows incoming connections on port 8080
  • The proxy only works while both devices are on the same network
  • If your PC's IP address changes, you'll need to update the proxy settings on your mobile device
  • Some apps may bypass system proxy settings - you may need to configure them individually
